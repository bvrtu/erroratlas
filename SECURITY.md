# Security policy

## Supported versions

Security fixes are provided for the latest released minor version.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting feature on the repository's **Security** tab. Include reproduction steps, impact, and any suggested mitigation.

ErrorAtlas scans source code locally and does not transmit source, error messages, or catalog contents. Reports may contain file paths and application error text, so review generated JSON or SARIF files before sharing them publicly.
